Welcome to apptanium mail

Email is awful. The protocols that underpin email today are wholly deficient for the needs that email-like systems should fulfil.

The apptanium-mail project is an effort to reimagine email for the new era of the always-on, always-connected world.

Apptanium-mail aims to create a new, open platform for mail that focuses on modern messaging and communication workflows. The primary goals of a relevant messaging system are:

  • In-grained security: the messaging system should seamlessly use a key exchange system between two parties to guarantee security and integrity of the message in transit
  • Identity: email addresses have become de-facto unique identifiers for individuals on the internet. A modern mail system should provide strong identity services so that one's email address is usable as a universal identifier instead of being brokered via other services
  • Trust: to complement identity services, the email system should also provide an infrastructure of trust services by providing access to message verification, etc. A mail system that embodies trust and identity intrinsically has the following benefits:
    • Prevent Phishing attacks and spam
    • Effective blocking from email subscription lists
    • A comprehensive handshaking process between two parties to seamlessly ensure secure, trusted communications
  • Social: the system allows social graph connections from other email systems following the ng-mail protocol. For instance, user alice@abc.com with a server on one system should be able to establish a social graph connection with user bob@bcd.com on another server.
  • Apps and APIs: the messaging system should be built to natively handle connections from apps and through APIs. Utilization of techniques like OAuth, etc. should be used to facilitate this functionality. The system should also allow the incorporation of various plugins to enhance core functionality.
  • Strong Auditing: provide blockchain based audit trails to enable strong auditing of email systems where such compliance is required
  • Modern Deployment Options: the server system should be compatible with modern cloud-centric technologies, such as containers, etc. so that installations can be managed, secured, and maintained with greater ease.

How it works

Just as current mail servers are installed on a company's servers, or on a hosting platform in a multi-tenant environment, the ng-mail server can be installed on-premises by a company or an enthusiast, and can also be operated by a hosting platform in a multi-tenant environment.
